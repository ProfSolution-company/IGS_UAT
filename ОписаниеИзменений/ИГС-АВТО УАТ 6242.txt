Внешняя печатная форма "Пакетная выписка путевых листов (УАТ)"
             
    Реквизиты обработки:
        Добавлены новые реквизиты:
             СсылкаНаОбъект новый реквизит, тип ДокументСсылка.уатПутевойЛист
    Макеты:
        Добавлены новые макеты:
            _3 - макет печатной формы для вида ПЛ 3(легковой)
            _3спец - макет печатной формы для вида ПЛ 3С (спец. автомобиля)
            _4П - макет печатной формы для вида ПЛ 4П (повременный)
            _6С - макет печатной формы для вида ПЛ 6С (автобус необщего пользования)
            _3_новый - макет печатной формы для вида ПЛ 3(легковой) если это ИГСАвто
            _3спец_новый - макет печатной формы для вида ПЛ 3С (спец. автомобиля) если это ИГСАвто
            _4П_новый - макет печатной формы для вида ПЛ 4П (повременный) если это ИГСАвто
            _6С_новый - макет печатной формы для вида ПЛ 6С (автобус необщего пользования) если это ИГСАвто
            _МаршрутныйЛист - макет печатной формы для печати маршрутного листа
            Эталон - макет печатной формы для печати штрихкодов
    Формы:
        Добавлена основная форма обработки:
            Реквизиты формы:
                СсылкаНаДокумент - ссылка на документ, на основании которого будет сформирована печатная форма
            Элементы формы:
                СсылкаНаДокумент - элемент формы ссылка на документ, на основании которого будет сформирована печатная форма
                ТабДок - элемент формы  предпросмотра печатной фомры
            Команды формы:
                ПечатьЛицевой - команда печати лицевой стороны путевого листа
                ПечатьОборотной - команда печати оборотной стороны путевого листа
                ПечатьПутевой - команда печати и оборотной и лицевой стороны путевого листа
                ПечатьПутевойМаршрутный - команда печати и оборотной и лицевой стороны путевого листа совместно с маршрутным листом 
            Модуль формы:
                ПечатьПутевойНаСервере() - метод для обработки команды печати на сервере
                ПечатьПутевой(Команда) - метод для обработки команды печати на клиенте
                ПечатьПутевойМаршрутныйНаСервере() -  метод для обработки команды печати на сервере для маршрутного листа
                ПечатьПутевойМаршрутный(Команда) - метод для обработки команды печати на клиенте для маршрутного листа
                ПечатьЛицевойНаСервере() - метод для обработки команды печати лицевой стороны путевого листа на сервере
                ПечатьЛицевой(Команда) - метод для обработки команды печати лицевой стороны путевого листа на клиенте
                ПечатьОборотнойНаСервере() - метод для обработки команды печати оборотной стороны путевого листа на сервере
                ПечатьОборотной(Команда) - метод для обработки команды печати оборотной стороны путевого листа на клиенте
    Модуль объекта:
        Глобальные переменные:
            ПечатьЧекЛиста - переменная отвечает за отображение или отсутствие чек листа, имеет тип данных Булево
            МаршрутныйЛист - переменная отвечает за отображение или отсутствие маршрутного листа, имеет тип данных Булево
            ДополнительныеДанныеВШапке - переменная отвечает за отображение дополнительных данных в шапке печатной формы
            ЭтоИГСАвто - переменная отвечает за определение принадлежности документа путевой лист к организации ИГС Авто, имеет тип данных Булево и определяет какой макет использовать для печати новый или старый
            ДанныеДляПечати - Переменная отвечает за хранение всех данных, которые необходимы для печати, имеет тип данных структура
            НаименованиеКонтрагентаСокращенное - Переменная отвечает за хранение информации о сокращенном наименовании контрагента
            КомпонентаПечатиШК - Переменная отвечает за хранение информации о штрихкоде
            рсТранспортноеСредство - Переменная отвечает за хранении информации о транспортном средстве
            ОбъектПечати - Переменная отвечает за хранение информации о Путевом листе и его табличных частях
        Методы:
            СведенияОВнешнейОбработке() - Метод отвечает за заполнение информации об обработке для её регистрации в системе
            ПолучитьНазначениеОбработки() - Метод отвечает за добавление печатной форы в список печатных форм документа уатПутевойЛист
            ДобавитьКоманду(ТаблицаКоманд, Представление, Идентификатор, Использование, ПоказыватьОповещение = Ложь, Модификатор = "") - Метод отвечает за добавление команд для вызова печатных форм
            Печать(МассивОбъектов, КоллекцияПечатныхФорм, ОбъектыПечати, ПараметрыВывода) - Метод отвечает за непосредственно запуск формирования печатной формы документа и задает условие выбора печатной формы
            ЗакэшироватьНеобходимыеДанные(СсылкаНаОбъект) - Метод отвечает за сохранение данных документа по его ссылке на весь период работы метода Печать
            СформироватьТабличныйДокумент(МассивОбъектов, КоллекцияПечатныхФорм) - Метод формирует табличных документ для дальнейшей печати
            ЗаполнитьДанныеДляПечати(СсылкаНаОбъект) - Метод отвечает за заполнение данных для печати
            СформироватьНаименованиеОрганизации(Орг) - Метод отвечает за формирование наименования организации
            уатПечать_3(ИмяМакета, ФлагПечати="") - Метод отвечает за формирование объекта печати для макетов "_3" и "_3_новый"
            уатПечать_3Спец(ИмяМакета, ФлагПечати="") - Метод отвечает за формирование объекта печати для макетов "_3спец" и "_3спец_новый"
            уатПечать_4П(ИмяМакета, ФлагПечати="") - Метод отвечает за формирование объекта печати для макетов "_4П_" и "_4П_новый"
            уатПечать_6С(ИмяМакета, ФлагПечати="") - Метод отвечает за формирование объекта печати для макетов "_6С" и "_6С_новый" 
            ПечатьМаршрутногоЛиста(ТабличныйДокумент, Макет) - Метод отвечает за печать маршрутного листа
            ПечатьМаршрутногоЛиста_Новый(ТабличныйДокумент) - Метод отвечает за печать маршрутного листа если это ИГС Авто
            ПолучитьШтрихкод() - Метод отвечает за формирование штрихкода
            ПолучитьКартинкуШтрихкода(ВнешняяКомпонента, ПараметрыШтрихкода) - Метод отвечает за получение картинки штрихкода
            ОписаниеОрганизации(СписокСведений, Список = "", СПрефиксом = Истина) - Метод отвечает за описание организации по её сведениям
            ПредставлениеАдреса(ВхАдрес) - Метод отвечает за форматирование адреса назначения и адреса отправления
            СведенияОЮрФизЛице(ЮрФизЛицо, ДатаПериода = '20200101', ДляФизЛицаТолькоИнициалы = Истина, БанковскийСчет = Неопределено) - метод пердоставляет сведения о физическом или юридическом лице
            ПолучитьАдресИзКонтактнойИнформации(Объект, ТипАдреса = "") - Метод позволяет из объекта контактной информации получить адрес
            ПолучитьТелефонИзКонтактнойИнформации(Объект) - Метод позволяет из объекта контактной информации получить телефон
            ПолучитьУдостоверениеСотрудника(Дата, ФизЛицо, ВидДокумента) - Метод позволяет получить удостоверение сотрудника по физ лицу
            уатПрочитатьРеквизитыТС(ТС, Реквизит = "", ДатаСреза = Неопределено, СведенияЕсть = Истина, флСообщать = Ложь) - Метод отвечает за 
получение дополнительных реквизитов
